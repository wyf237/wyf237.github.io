位运算

1. 什么是位运算

数据在计算机中是以二进制数的形式存放的。
a = 01101001B
b = 01011110B

- 高 低

2. 位运算有哪些

  • 与（and）、或（or）、取反（not）、异或（xor）、左移、右移
  • 按位“与”、按位“或”、按位“取反”、按位“异或”

按位“与”

- - - - -
ai 0 0 1 1
bi 0 1 0 1
结果 0 0 0 1

按位“或”

- - - - -
ai 0 0 1 1
bi 0 1 0 1
结果 0 1 1 1

按位“异或”

- - - - -
ai 0 0 1 1
bi 0 1 0 1
结果 0 1 1 0

按位“取反”

- - -
ai 0 1
结果 1 0

移位运算

移位：将二进制数的每一位向左或向右移动指定的位数

如：左移两位

内存位置 外面 外面 7 6 5 4 3 2 1 0
移动前 0 1 1 0 1 0 0 1
移动后 0 1 1 0 1 0 0 1 0 0

如：右移两位

内存位置 7 6 5 4 3 2 1 0 外面 外面
移动前 0 1 1 0 1 0 0 1
移动后 0 0 0 1 1 0 1 0 0 1

3. 位运算符

  • & 按位“与”
  • | 按位“或”
  • ^ 按位“异或”
  • << 左移
  • >> 右移

  • 格式
    <整数类型表达式1> <位运算符> <整数类型表达式2>
    ~<整数类型表达式>

6. 位运算的应用

按位“与”可以将某些位设置为0
(a=0110 1001B) & (b=1111 0000B) = (c=0110 0000B)

按位“或”可以将某些位设置为1
(a=0110 1001B) | (b=1111 0000B) = (c=1111 0000B)

按位“异或”可以让某些位变反，而另一些位不变
(a=0110 1001B) ^ (b=1111 0000B) = (c=1001 1001B)

7. 复合位运算

  1. 位运算可以使用复合运算符
    • 复合与：&=
    • 复合或：|=
    • 复合异或：^=
    • 复合左移：<<=
    • 复合右移：>>=
  2. int x=0x1A6F,y=0x35AA,k=3:
    • x&y=; 等价于 x=x&y;
    • x|y=; 等价于 x=x|y;
    • x^y=; 等价于 x=x^y;
    • x<<=l; 等价于 x=x<<k;
    • x>>=k; 等价于 x=x>>k;
    • 优先级：很低，与复合算数运算、等号同级
    • 结合顺序：从右向左

8. 位运算使用注意事项

  1. 一个&和两个&&是不同的
    &是按位“与”，&&是逻辑“与”
    |是按位“或”，||是逻辑“或”
  2. 在输出中使用移位运算的表达式时，要加括号

逻辑运算

1. 逻辑运算

  • 逻辑运算表达、判断多个条件之间的关系
    同时成立、之一成立、不成立
  • 例如
    • $x\in[0,1]$，可以表示为
      x >= 0 并且 x <= 1
    • $x\notin[0,1]$，可以表示为
      x > 1 或者 x < 0
  • 非负实数
    不是(x < 0)，非(x < 0)
  • “并且”，“或”，“不是”，就是逻辑运算

2. 逻辑运算符

表示逻辑运算的符号

  • &&——并且，逻辑“与”，and
  • ||——或，逻辑“或”，or
  • !——不是，逻辑“非”，not

3. 逻辑表达式

  • 逻辑运算符和关系表达式或有数值的表达式连接构成逻辑表达式
  • &&，||是双目运算，格式是
    <条件1> && <条件2>
    <条件1> || <条件2>
  • !是弹幕运算，格式是
    !<条件>
  • 例如

4. 逻辑表达式的值

逻辑值：true，false

  • 与运算 <条件1> && <条件2>

    条件1条件2结果
    true true true
    true false false
    false true false
    false false false

  • 或运算 <条件1> || <条件2>

    条件1条件2结果
    true true true
    true false true
    false true true
    false false false

  • 非运算 !

    条件结果
    true false
    false true

  • 举例

    逻辑表达式 -0.5 0.5 5
    x >= 0 && x <= 1< td> false true false
    x >= 0 || x <= 1< td> true false true
    !(x >= 0 && x <= 1)< td> true false true
    x < 0 true false false
    !(x < 0) false true true

6. 注意事项

  1. 单个变量或常量也可以作为条件
    1 && 2 ——— true
    1 && 0 ——— false
    0 || 1 ——— true
    0 || 0 ——— false
    !2 ——— false
    !0 ——— true
    a && b ——— a!=0 && b!=0
    a || b ——— a!=0 || b!=0
    !a ——— a==0，a为0时，!a为true
    a ——— a!=0
  2. 逻辑表达式的短路求值（short circuit evaluation）
    • 如果判断了前面的条件，已经能达到表达式的结果，就不再计算后面的条件
      如x>=0 && x<=1，如果x=-1，x>=0结果为false，那么此表达式的值必然为false，那么x<=1这里就不再计算
    • 逻辑运算符也用在条件分支语句当中

  • C++中用+，-，*，/，%分别表示加、减、乘、除、求余运算，例如：

  • 注意事项
    1. 乘法运算符不能省略
      数学中写$b^2-4ac$，程序中应写为$bb-4.0a*c$,不可以写成$bb-4ac$
    2. 除数是表达式时注意加括号，例如：
      $\dfrac{x}{2a}$在C++中应写为：x/2.0/a 或 x/(2.0a)
      不可以写成x/2      a 或 x/2a
  • 其他运算符的实现
    • C++中没有乘方运算符，也没有指数、对数、三角函数的运算符，它们是通过函数的调用实现的。如：
      pow(x,n)——————$x^n$
      exp(x)———————$e^x$
      log(x)———————$log_ex$
      log10(x)——————$lg_x$
      sin(x)———————$sinx$（弧度）
      cos(x)———————$cos(x)$（弧度）
      tan(x)———————$tanx$（弧度）
      1/tan(x)——————$cotx$（弧度）
      asin(x)——————-$x\in[-1,1]$

3. 算数运算符的优先级

  • 先乘除求余，后加减，级别相同的从左向右逐个计算
    $1+2*3, 2+3+4，6*2/3$
  • 圆括号可以改变优先级
    $(1+2)*3$
  • 圆括号可以嵌套，越靠里，圆括号的优先级越高
    $((1+2)3+(2+3)2)*7$
  • %和*，/优先级别相同
  • 后置++、—的优先级高于前置++，—的优先级
  • 前置++，—的优先级高于乘除的优先级

4. 运算的数据类型

  1. 相同类型的数据进行算数运算，结果还是这种数据类型
    • 1 + 2，1 - 2，1 * 2，1 / 2（1 / 2的结果是0而不是0.5）
    • 1.0 + 2.0，1.0 - 2.0，1.0 * 2.0，1.0 / 2.0（1.0/2.0的结果0.5）
  2. 不同类型的数据运算，默认向级别高的类型转换
    （低） short，char → int → unsigned → long → double （高）
    char + int → int
    int + double → double

  3. 强制类型转换

    • (<类型>)x 或 <类型>(x)
    • 将x转换为<类型>的数据

    • x可以是表达式

    • 高精度 → 低精度，会损失有效数字

5. 复合运算符

  • 为简化书写，C++提供复合算数运算符
    +=，-=，*=，/=，%=
  • 双目运算符
    a += b 等价于 a = a + b
    a -= b 等价于 a = a - b
    a = b 等价于 a = a b
    a /= b 等价于 a = a / b
    a %= b 等价于 a = a % b

第一讲 什么是算法

一、算法与问题求解的基本概念

算法：是解决问题的方法或者过程，严格地讲是满足下述性质的指令序列：

  • 输入：有0个或多个外部量作为算法的输入；
  • 输出：算法产生至少1个量作为输出；
  • 确定性：组成算法的每条指令清晰、无歧义；
  • 有限性：算法中每条指令的执行次数有限，执行每条指令的时间也有限。

程序：是算法用某种程序设计语言的具体实现。程序可以不满足算法的性质（4）即有局限性。

二、算法时间复杂度的表示方法以及分析技术

算法初体验1——“玩具”问题

问题描述：判断一个byte（无符号整数）里面有多少个bit的值为1？

  • 如果这个子函数需要调用多次，内存足够，如何提高性能？

    采取打表，把所有的无符号整数有多少个bit值为1的答案算出来存到数组中，然后从数组中去寻找答案， 更省时间。

  • 如果既想省时间，又想省空间，怎样改进？

    我们可以应用一个数据结构：哈希表，在应用这个数据结构的时候，需要iValue这个答案，先去哈希表里去找，如果有答案，那么返回，如果没有现成的答案，那么调用计算的算法，然后返回答案。这样就相对既省时间，也省空间。

  • 总之：时间和控件是算法的度量标杆！
