【C++基础】074结构化和面向对象程序设计思想

发表于 2019-11-13 分类于 C++基础

简介：结构化和面向对象程序设计思想

两种程序设计思想

  1. 面向过程的程序设计（Structured Programming）

    • 以功能为中心，通过分解功能，采用函数来描述操作
    • 数据与函数分离，数据（类型或结构）一旦发生变化，函数也要相应改变
    • 例如排序函数：void sort(int a[],int n);只能排序整数数组

  2. 面向对象程序设计（Object Oriented Programming）

    • 以数据为中心，采用对象来描述颞部属性和操作方法
    • 将数据和函数当作一个统一体，采用软件对象模拟可观世界中的实际对象
    • 例如：钟表类，日期类

OOP方法特征

  1. 抽象： 实体特征+实体功能
    如钟表、日期、分数等实体的抽象描述
  2. 封装： 数据和函数封装组成统一体
    通过公有成员函数修改数据、显示数据、取得数据
  3. 继承： 将对象（实体）之间的差异通过继承性加以抽象描述
    动物→｛人、大象、鸟……｝
    人→｛学生、工人、士兵……｝
    学生→｛大学生、中学生……｝→……
  4. 多态： 指相同语法结构（数据或操作）代表多种功能
    如：+、-、*、/等

OOP方法的优点

  1. 开发时间短、效率高、可靠性高
  2. 重用、共享、可维护性、精简
  3. 适合大型程序长时间的团队开发工作

为什么要引入

  1. 为什么要引入数组？
    太多的变量（全班同学的C++成绩）

  2. 为什么要引入函数？
    结构化程序设计：功能分解，代码重用

  3. 为什么要引入指针？
    加快速度、参数传递、动态分配内存

  4. 为什么要引入结构体
    复合数据结构，不同数据类型的聚合在一起

  5. 为什么要引入类

    • 采用人们认识客观世界的自然方式去模拟可观世界中的对象
    • 提高开发程序的效率和可靠性

定义日期时间类的2种可能方式

  1. 重新完整定义日期时间类，代码重复

  2. 利用已存在的类定义日期时间类，代码复用
    这就是 继承性 抽象描述实体的基本思路。

日期时间类多态性设计思考

  1. 假设有2个日期时间的对象：mydatetime, yourdatetime
  2. 计算这2个对象之差：mydatetime - yourdatetime
  3. 结果是什么呢？相隔多少年？相隔多少日？相隔多少分？
  4. 毫无疑问这些结果都是有意义的
  5. 问题是能不能直接计算表达式：mydatetime - yourdatetime
  6. 减号两边是整数、浮点数、双精度数都可以计算
  7. 当然希望减号两边也可以是日期时间对象，这就是表达式的多态性
  8. 当然希望减号两边也可以是分数对象
