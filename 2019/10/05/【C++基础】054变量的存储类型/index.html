变量的存储类型

  2. 变量的存储类型
    变量的存储类型是在内存中存储的方式，根据变量的存储类型，可以知道变量的作用域和生存期。这里，介绍4个存储类型，分别是auto（自动型），register（寄存器类），static（静态类）和extern（外部类）。
    在C++中定一个变量的完整形式是：

    • 自动变量——用auto修饰（默认的定义方式）
      如：定义一个局部变量i。
      auto int i;与int i;是相同的。
      说明： 自动变量在其定义块（函数或符合语句）开始执行时分配空间，在块执行结束时释放空间。所以自动变量的生命周期开始于块的执行，终止于块的结束。
    • 寄存器变量——用register修饰
      将尽可能存放在CPU的寄存器中，以提高程序的运行效率
      注意： 仅局部变量和形参可作为寄存器变量。

    • 静态变量——用static修饰

      • 静态变量分配在全局数据区中，定义时系统将提供默认的初始值。
      • 静态变量在编译时分配存储控件，在整个程序执行结束后释放存储空间。所以静态变量具有全局生命周期。
      • 根据声明的位置不同，静态变量又分为 静态局部变量 和 静态全局变量 。
      • 静态局部变量是在“块”中定义的静态变量。它具有局部作用域，却有全局声明周期。在“块”执行结束后，该静态局部变量并不释放（其值依旧存在），以便下次调用时可继续使用。

      • 全局变量可以在其他源文件中使用。如果在全局变量前加上static修饰符，则成为静态全局变量。静态全局变量只能在本文件中使用。
        例如：函数调用计数器。使用静态局部变量统计某个函数被调用的次数。
